# personal-pasteboard Security Baseline

This project is deployed as a private service. It is not published directly to the internet.

## Required Runtime Settings

- `PASTEBOARD_PASSWORD_HASH`: generated by `node scripts/generate-secrets.js`.
- `PASTEBOARD_SESSION_SECRET`: base64 encoded 32-byte key for signed session cookies.
- `PASTEBOARD_DATA_KEY`: base64 encoded 32-byte key for AES-256-GCM data encryption.
- `PASTEBOARD_SECURE_COOKIES=false` is only acceptable for SSH tunnel or localhost access.
- Set `PASTEBOARD_SECURE_COOKIES=true` when the service is exposed through HTTPS.

## Deployment Rules

- Keep the host port bound to `127.0.0.1:3088`.
- Do not expose this app directly on a public interface.
- Access it through SSH tunneling, VPN, or a reverse proxy protected by HTTPS and IP allowlists.
- Keep `.env` mode `0600`.
- Keep `data/` mode `0700`.
- Keep `data/pastes.json.enc` mode `0600`.

## Container Baseline

The compose file intentionally enforces:

- non-root container user from the Dockerfile
- `read_only: true`
- writable `/tmp` only through tmpfs
- `cap_drop: [ALL]`
- `security_opt: [no-new-privileges:true]`
- health checks

## Backup Rule

Back up both `data/pastes.json.enc` and `.env`. The encrypted data file is not recoverable without `PASTEBOARD_DATA_KEY`.
